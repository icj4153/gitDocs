"초보자를 위한 git & gitHub 사용 설명 웹 페이지 개발"
초보자들은 github 사용법을 몰라서 어려움을 겪고, 단순히 ‘git add’, ‘git commit’, ‘git push’만 사용하는 경우가 많다. 또한, conflict가 생겼을 때 원인을 파악하지 못해서 directory를 삭제하고 다시 clone을 한다거나, 함부로 사용하면 위험한 ‘--force’까지 사용하는 경우가 있다. 이것은 git의 명령어와 github의 PR 등의 사용법을 제대로 모르는 상태로 프로젝트를 진행하기 때문이다. 이에 따라 본 프로젝트에서는 초보자들을 위한 git & gitHub 사용법을 알려주는 웹사이트를 개발하고자 한다. 더 나아가 gitHub으로 어떻게 협업이 이뤄지는 지 issue, PR 등의 다양한 기능들과 convention들을 알아보고 활용해본다.

협업 과정

페이지를 개발하기 위해 HTML, CSS, JS의 기초 문법들을 학습하고 GIT을 통해 협업을 진행하였다.
github에서는 페이지 내용을 작성하면서 얻은 지식을 활용하고자 branch, PR, Issue 기능을 다양하게 사용하였다.

Branch
Working on branch

- develop 브랜치 생성
- develop 브랜치에서 분기
-> 개인별로 하나의 브랜치를 생성
- 각자 브랜치에서 작업
- rebase, merge 사용하여 합침

PR
Working on PR

- 각자 브랜치에서 작업
- git add, git commit -m, git push 명령어 사용
- Pull Request 생성
- 나머지 팀원들 PR 검토
-> 승인 여부 결정

Issue
Working on Issue

- 각자 개인별 Issue 생성
- 각 Issue는 목적, 작업 상세 내용,
  참고 사항으로 구성
- 작업 상세 내용은 체크리스트 사용
