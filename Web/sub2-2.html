이번 글에서는 Github가 무엇인지, Github의 장점, 그리고 Github를 통해 수행할 수 있는 것에는 무엇이 있는지, Github의 기본적인 용어에는 어떤 것이 있는지에 대해 안내한다.

Github

Github란?

  • 분산 버전 컨트롤 소프트웨어 Git을 기반으로 소프트웨어 개발 및 버전 제어를 위한 인터넷 호스팅 서비스이다.
  • Git의 분산 버전 제어와 액세스 제어, 버그 추적, 소프트웨어 기능 요청, 작업 관리, 지속적인 통합을 제공한다.
  • 일반적으로 오픈소스 소프트웨어 개발 프로젝트를 호스팅하는 데 사용된다.
  • 표준 Git 명령줄 인터페이스를 사용하여 액세스하고 관리할 수 있다.
  • 사용자가 사이트에서 공개 리포지토리를 탐색할 수 있도록 한다.

Github의 장점

  • 원하는 시점으로 복원 가능
  • 작업한 내역을 타임라인 순으로 확인 가능
  • 수정한 내용에 대한 문서화 용이
  • 브랜치 생성 및 병합 용이
  • 여러 사람이 하나의 소스 작업 시 충돌 방지 및 해결
  • 누가 어떤 작업을 수행했는지 확인 가능

Github를 통해...

  • 원격 저장소에서 Git을 사용할 수 있다.
  • 지역 저장소를 백업할 수 있다.
  • 협업 프로젝트에서 사용할 수 있다.
  • 자신의 개발 이력을 남길 수 있다.
  • 다른 사람의 소스를 살펴볼 수 있고, 오픈 소스에 참여할 수도 있다.

Github 기본 용어

  • 저장소(Repository)
  • 프로젝트가 살고 있는 저장공간.
  • 주로 'repo'라고 줄여서 사용한다.
  • 이 저장소 안에 코드, 텍스트, 이미지를 저장할 수 있다.
  • 버전 관리(Version Control)
  • 팀 단위로 프로젝트를 운영할 때 필수적인 기능
  • 다수의 사용자가 하나의 페이지를 수정해도 각 사용자별로 수정 내용을 저장
  • 모든 시점의 '스냅 샷'을 저장하기 때문에 겹쳐 쓸 일이 없다.
  • 커밋(Commit)
  • 개발 단계별로 깃발을 꽂는 행위
  • 간단한 메모를 남겨서 이 지점까지 어떤개발을 했다는 표시를 할 수 있다.
  • 브랜치(Branch)
  • 자신이 진행하는 프로젝트의 브랜치를 따서 자기의 버전을 만드는 것
  • 작업을 마무리하면 프로젝트 메인 디렉토리인 master에 브랜치를 merge한다.
