브랜치(Branch)란?

소프트웨어를 개발할 때에 개발자들은 동일한 소스코드를 함께 공유하고 다룬다.
동일한 소스코드에서 어떤 개발자는 버그를 수정하고, 다른 개발자는 새로운 기능을 만들어 낸다.
여러 사람이 같은 소스코드에서 동시에 작업을 하면 서로 다른 버전의 코드가 만들어진다.
이 때, 여러 개발자들이 동시에 다양한 작업을 할 수 있게 만들어 주는 기능이 바로 브랜치이다.
브랜치는 각자 독립적인 공간을 만들어서 자유롭게 코드를 개발할 수 있게 해준다.
이렇게 각자 개발한 코드는 나중에 merge나 rebase 과정을 통해 하나의 코드로 합쳐진다.

브랜치 생성방법

1. 브랜치 확인하기
  1) 현재 내가 위치한 branch 확인
      - git branch
      - git status
  2) 원격 저장소의 브랜치 확인
      - git branch -r
  3) 브랜치의 마지막 커밋 메세지 확인
      - git branch -v
2. 브랜치 생성 및 이동
  2.1 첫번째 방법
  1) Branch 생성하기
      - git branch [브랜치명]
  2) 생성한 Branch로 이동하기
      - git checkout [브랜치명]
      - git switch [브랜치명]
  2.12 두번째 방법
  1) 브랜치를 새로 만들면서 브랜치 변경
      - git checkout -b [브랜치명]
      - git switch -c [브랜치명]
3. branch 삭제
      - git branch -d [브랜치명]
      - git branch -D [브랜치명]
      - git push origin --delete [브랜치명]
