깃허브(GitHub)로 협업하기 : PR(Pull Request)란?

이번 글에서는 깃허브(Github)로 협업할 때 사용하는 PR(Pull Request)에 대해서 설명한다.

1. Pull Request란?

github repository branch에 push한 내역들, 변경 사항을 다른 사람들에게 알리는 것을 의미한다.
main branch에 merge 전, 변경 사항에 대해 다른 사람들과 논의, 검토할 수 있다.
다른 사람의 commit을 볼 일이 많지 않고 main branch merge에서야 확인하게 되는데, PR은 당장 merge하지 않는다는 규칙으로, 코드에 신경 쓰게 되고 어떤 작업이 언제 적용되었는 지 확인할 수 있다.
따라서 코드 충돌(conflict)을 줄일 수 있다.

  1. code review를 할 수 있다.

    2. main branch에 merge하기 전, 변경 사항에 대해 다른 사람들과 논의 및 검토할 수 있다.

  2. 충돌(conflict) 방지할 수 있다.

    3. 다른 사람의 commit을 볼 일이 많지 않고 main branch에 merge할 때 되어서야 확인하는 경우가 많다.
    PR은 당장 merge 하지 않는다는 규칙으로, 코드에 신경 쓰게 되고 어떤 작업이 언제 적용되었는 지 확인할 수 있다.

2. PR(Pull Request) 사용법

작업한 branch에 push한 이후 합치고자 하는 branch로 PR을 올린다.
그리고 code review를 진행하고 이상이 없다면 merge한다.

  1. PR 생성하기 전

    2. 분기한 branch에서 작업한 내용들을 git add, git commit한다.
    그리고 분기한 branch에 push한다.

  2. PR 생성하기

    3. Compare & pull request
    base branch, compare branch를 선택한다.
    내용을 작성한다.
    Create pull request
    PR 생성 후에 수정사항이 생겨도 같은 branch에 push하면 자동으로 반영된다.

  3. Code Review

    4. PR review에서 comment를 달 수 있다.
    Files changed 탭에 들어가서 한 줄마다 comment 등록도 가능하다.

  4. Merge

    5. 최종적으로 승인하는 단계이다.
    Merge pull request -> Confirm merge를 한다.

  5. Delete branch

    6. 해당 브랜치 개발이 완료되었으면 브랜치를 삭제한다.
